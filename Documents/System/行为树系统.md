## ID 分段规则

- **TTT**：业务大类（001-999）
- **B**：节点类型（1=条件，2=行为，3=顺序，4=选择，5=反转）
- **SSS**：序列号（001-999）

## 命名规范

### 基础规则

| 节点类型 | cid命名规则 |
|----------|------------|
| Condition | `是否…` / `目标是否…` |
| Inverter | `是否不…` |
| Action | `…行为`|
| Sequence | `自动…` / `…组` / `…条件` |
| Selector | 业务名词 / `…追踪` |

### cid唯一性约束

**硬性规则：** 所有节点的cid必须全局唯一，不允许任何重复。

### 层次命名一致性

同一业务系统的节点应保持命名一致性：

```
业务名(Selector)
├─ 自动业务名A(Sequence)
│  ├─ 业务名A目标是否存在(Condition)
│  └─ 业务名A追踪(Selector)
│     ├─ 业务名A组(Sequence)
│     │  ├─ 业务名A目标是否相邻(Condition)
│     │  └─ 业务名A行为(Action)
│     └─ 自动寻路业务名A目标(Action)
```

## 固定范式
这些范式广泛应用，实现高复用与统一的执行流程。

### 顶层分支（Selector）

**定义：** 将多个相关业务模块组合成独立角色的顶层选择器。

```
顶层分支 \[Selector]
├─ 子业务A \[Selector/Sequence]
├─ 子业务B \[Selector/Sequence]
└─ 子业务C \[Selector/Sequence]
```

**典型节点：** `24001 经济`、`24002 店主`、`44001 农夫`、`34003 猎人`

**设计原则：**
- 按优先级排列子业务
- 子业务之间互斥，选择第一个成功的执行

---

### 策略选择（Selector）

**定义：** 将同一业务目标的多种实现策略组织为选择器，按优先级尝试。

```
策略选择 \[Selector]
├─ 自动策略A \[Sequence]  - 优先级高，条件严格
├─ 自动策略B \[Sequence]  - 优先级中
└─ 自动策略C \[Sequence]  - 优先级低，兜底
```

**典型节点：** `34001 狩猎`、`24006 经营`

**设计原则：**
- 优先选择成本低、收益高的策略
- 最后一个作为兜底策略

---

### 自动（Sequence）

**定义：** 完整的自动化行为流程，包含目标检测、追踪与执行。

```
自动 \[Sequence]
├─ 目标是否存在 \[Condition]
└─ 追踪 \[Selector]
   ├─ 行为组 \[Sequence]
   │   ├─ 目标是否相邻 \[Condition]
   │   └─ 行为 \[Action]
   └─ 自动寻路目标 \[Action]
```

**典型节点：** `33001 自动狩猎`、`23001 自动清仓`、`43009 自动采集`

---

### 蓄意追踪（Selector）

**定义：** 先尝试就地执行，失败时自动寻路到目标位置。

```
蓄意追踪 \[Selector]
├─ 就地行为组 \[Sequence]
│   ├─ 目标是否相邻 \[Condition]
│   └─ 行为 \[Action]
└─ 自动寻路目标 \[Action]
```

**典型节点：** `34002 狩猎追踪`、`24003 清仓追踪`

---

### 条件行为组（Sequence）

**定义：** 最小执行单元，检查条件后执行行为。

```
条件行为组 \[Sequence]
├─ 条件 \[Condition]
└─ 行为 \[Action]
```

**典型节点：** `33002 狩猎组`、`23003 清仓组`

---

### 一次性行为（Selector）

**定义：** 玩家专用范式，达成目标后自动清除行为树。

```
一次性行为 \[Selector]
├─ 清除组 \[Sequence]
│   ├─ 是否达成 \[Condition]
│   └─ 清除行为树 \[Action]
└─ 行为 \[Action]
```

**典型节点：** `0064001 一次性寻路`

---

### 完整层次关系示例

```
顶层角色 \[Selector] ← 顶层分支
├─ 业务模块A \[Selector] ← 顶层分支
│  ├─ 策略A1 \[Selector] ← 策略选择
│  │  ├─ 自动XXX \[Sequence] ← 自动
│  │  │  ├─ XXX目标是否存在 \[Condition]
│  │  │  └─ XXX追踪 \[Selector] ← 蓄意追踪
│  │  │     ├─ XXX组 \[Sequence] ← 条件行为组
│  │  │     │  ├─ XXX目标是否相邻 \[Condition]
│  │  │     │  └─ XXX行为 \[Action]
│  │  │     └─ 自动寻路XXX目标 \[Action]
│  │  └─ 自动YYY \[Sequence]
│  └─ 策略A2 \[Selector]
└─ 业务模块B \[Selector]
```

## 组合规则
* 前置条件严格：所有 Action 前必须有条件护栏（位置、点可用、目标存在与类型检查）。
* 无循环引用：所有复合节点必须保持单向、无环。

## 执行间隔数值

行为树执行间隔基于敏捷属性和节点修正值动态计算。

### 基础公式

$$
\text{BaseInterval}(\text{Agi}) = \frac{100000}{\text{Agi} + 100}
$$

### 实际间隔

$$
\text{ActualInterval} = \text{BaseInterval} \times \text{NodeMultiplier}
$$

- **BaseInterval**：基于敏捷计算的基础间隔（毫秒）
- **NodeMultiplier**：上次执行的行为树节点的修正系数（默认1.0）

### 节点修正机制

每个行为树节点可配置独立的 `interval_multiplier`，根据行为类型调整执行频率：

| 修正系数 | 适用场景 | 示例行为 |
|---------|---------|---------|
| 0.5 | 极高频响应 | 战斗反击、闪避判断 |
| 1.0 | 标准频率 | 寻路、追踪、逃跑、送货 |
| 2.0 | 中频决策 | 觅食、采集、拾取 |
| 3.0 | 中低频 | 补给、取货、存储 |
| 5.0 | 低频操作 | 整理、清仓、回家 |
| 8.0 | 极低频 | 烹饪、酿造、锻造、缝纫 |
| 10.0 | 超低频 | 巡逻（配合概率节点） |

**设计优势：**
- 保留敏捷属性的个性化影响（高敏捷角色仍比低敏捷快）
- 根据行为类型智能调频（低频行为自动降频，高频行为保持原速）
- 同一生物在不同行为状态下自动调整决策频率

### 基础间隔表

推导过程：

$$
\text{speed} = \frac{\text{Agi} / 100}{\text{Agi} / 100 + 1} = \frac{\text{Agi}}{\text{Agi} + 100}
$$

$$
\text{Interval} = 1000 \times (1 - \text{speed}) = 1000 \times \frac{100}{\text{Agi} + 100} = \frac{100000}{\text{Agi} + 100}
$$

- \( \text{Agi} \) 为敏捷属性值
- \( \text{speed} \) 为归一化速度系数，范围 \[0, 1)
- \( \text{Interval} \) 为执行间隔（毫秒），范围 (0, 1000\]

## 扩展与维护

1. **新增节点：** 优先复用现有范式，严格遵循命名规范，确保cid全局唯一。
2. **新增业务：** 同步更新ID分段规则表，为新业务分配专用编码范围。
3. **删除节点：** 必须检查所有引用，同步更新引用链，防止失效引用。
4. **修改cid：** 必须同步修改所有引用该cid的节点的nodes字段。
5. **Action命名：** 所有新增Action必须使用"动宾结构+行为"格式，无例外。
