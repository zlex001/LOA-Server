# 新手引导系统

**新手引导系统**（Guide System）负责为新玩家提供触发式引导，帮助玩家理解核心玩法、完成生存循环、感知付费价值。

本系统采用服务端驱动的触发式设计，根据玩家状态和行为触发相应引导，通过协议推送给客户端展示。

---

## Tip 体系 | Tip Family

**GuideTip** 是 Tip 体系的一个子类，与 **FlyTip**、**NumericalTip** 平级：

| 类型 | 用途 | 可重复 | 触发方式 |
|------|------|--------|----------|
| **FlyTip** | 飞字提示（升级、获得经验、状态提示） | 是 | 程序主动发送 |
| **NumericalTip** | 数字变化动画（数值从 start 到 end） | 是 | 程序主动发送 |
| **GuideTip** | 新手引导提示（教学、功能说明） | **否** | **条件触发** |

**GuideTip** 的特殊之处：
- 需要策划表配置触发条件
- 需要玩家存档记录已触发状态
- 需要触发系统监听事件并检查条件

## 引导设计矩阵 | Design Matrix

引导内容按**层次**分类，每个内容选择合适的**引导方式**实现：

### 引导方式分类

| 方式 | 英文 | 说明 | 实现手段 |
|------|------|------|----------|
| **显性引导** | Explicit | 直接告诉玩家 | GuideTip、NPC 对话、教程弹窗 |
| **隐性引导** | Implicit | 通过设计让玩家自然学会 | 关卡设计、视觉引导、难度曲线 |
| **涌现引导** | Emergent | 玩家自行试错学习 | 死亡后理解、重复尝试 |

### 内容层次 × 引导方式

| 层次 | 内容 | 显性引导 | 隐性引导 | 涌现引导 |
|------|------|----------|----------|----------|
| **操作层** | 移动（走格子） | - | 只有2格的新手场景 | - |
| **操作层** | 交互（点击NPC） | - | 必经之路有NPC挡住 | - |
| **操作层** | 战斗（攻击敌人） | - | 狭窄通道中的敌人 | - |
| **操作层** | 拾取（捡物品） | - | 敌人掉落发光物品 | - |

| **系统层** | Lp 机制 | GuideTip: Lp 低于阈值 | 饿晕后复活体验 | 试错理解 |

## 体验设计 | NPE

**NPE**（New Player Experience，新玩家体验）定义新玩家前7天的体验旅程，确保玩家理解核心玩法、体验游戏价值、到达转化节点。

### 生命周期概览

| 阶段 | 时长 | 目标 | 关键指标 |
|------|------|------|----------|
| **Survival** | 0-2小时 | 理解核心生存循环 | 引导完成率 |
| **Discovery** | 2-8小时 | 体验战斗与市场 | D1留存 |
| **Conversion** | 第2-3天 | 首充决策 | 首充率 |
| **Mastery** | 第4-7天 | 核心循环掌握 | D7留存 |
| **Commitment** | 第7天+ | 月卡订阅 | 付费率 |

### 第1天：生存阶段

**0-0.5小时：首次接触**

| 维度 | 内容 |
|------|------|
| 体验目标 | 理解 **Lp**（Life Points）决定生存 |
| 核心行为 | 进食以恢复 **Lp** |
| 系统触点 | **Lp** 系统、背包 |
| 触发条件 | **Lp** 首次低于50% |
| 引导类型 | 系统提示 |
| 流失风险 | 玩家饿死但不知道原因 |

**0.5-1小时：首次战斗**

| 维度 | 内容 |
|------|------|
| 体验目标 | 完成首次战斗并获得经验 |
| 核心行为 | 攻击敌人、获取战利品 |
| 系统触点 | 战斗系统、经验系统 |
| 触发条件 | 首次遭遇敌人 |
| 引导类型 | 上下文提示 |
| 流失风险 | 玩家不理解战斗操作 |

**1-2小时：市场入门**

| 维度 | 内容 |
|------|------|
| 体验目标 | 理解如何从市场获取食物 |
| 核心行为 | 从NPC商店购买食物 |
| 系统触点 | 商店系统、货币系统 |
| 触发条件 | 背包无食物且 **Lp** < 50% |
| 引导类型 | 系统提示 + 路径点 |
| 流失风险 | 玩家知道要吃但找不到食物来源 |

### 第1天：发现阶段

**2-8小时：核心循环建立**

| 维度 | 内容 |
|------|------|
| 体验目标 | 体验完整循环：战斗→拾取→出售→购买→进食→战斗 |
| 核心行为 | 在市场出售战利品、管理背包 |
| 系统触点 | 市场系统、背包管理 |
| 触发条件 | 背包容量 >= 80% |
| 引导类型 | 上下文提示 |
| 流失风险 | 背包满导致玩家受挫 |

### 第2-3天：转化阶段

**首充触点**

| 维度 | 内容 |
|------|------|
| 体验目标 | 感知付费便利性的价值 |
| 核心行为 | 查看首充优惠 |
| 系统触点 | 商城系统、充值币 |
| 触发条件 | 达到5级 或 游玩2小时 |
| 引导类型 | 软推广 |
| 转化目标 | 首充率 >= 15% |

**月卡价值展示**

| 维度 | 内容 |
|------|------|
| 体验目标 | 理解时间价值主张 |
| 核心行为 | 查看月卡权益 |
| 系统触点 | 订阅系统、行为树系统 |
| 触发条件 | 达到10级 或 游玩8小时 |
| 引导类型 | 功能介绍 |
| 转化目标 | 月卡购买考虑 |

### 第4-7天：掌握阶段

**行为树入门（月卡用户）**

| 维度 | 内容 |
|------|------|
| 体验目标 | 成功配置并运行行为树 |
| 核心行为 | 设置基础自动战斗 |
| 系统触点 | 行为树系统 |
| 触发条件 | 月卡激活 |
| 引导类型 | 交互式教程 |
| 留存目标 | D7留存 >= 30% |

### GuideTip 协议

**GuideTip** 是新增的引导提示协议，与 FlyTip/NumericalTip 平级：

**服务端→客户端：GuideTip**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int | 是 | 唯一引导ID，用于跟踪 |
| type | int | 是 | **GuideType** 枚举值 |
| title | string | 否 | 本地化标题 |
| content | string | 是 | 本地化内容 |
| action | object | 否 | 操作按钮配置 |
| waypoint | int[] | 否 | 地图坐标 [x, y, z] |

**GuideType** 枚举：

| 值 | 名称 | 说明 |
|----|------|------|
| 0 | SystemPrompt | 模态对话框，需要关闭 |
| 1 | ContextualHint | 非阻塞浮动文本 |
| 2 | Waypoint | 地图上的可视标记 |
| 3 | Achievement | 奖励通知弹窗 |
| 4 | SoftPromotion | 价值主张展示 |
| 5 | Tutorial | 交互式分步指引 |

**客户端→服务端：GuideResponse**

| 字段 | 类型 | 必填 | 说明 |
|------|------|------|------|
| id | int | 是 | 响应的引导ID |
| status | int | 是 | 响应状态枚举值 |

**Status** 枚举：

| 值 | 名称 | 说明 |
|----|------|------|
| 0 | Completed | 玩家完成/确认引导 |
| 1 | Dismissed | 玩家手动关闭 |
| 2 | ActionTaken | 玩家点击操作按钮 |

## 状态管理 | State

### 玩家引导状态

```csharp
public class PlayerGuideState
{
    // Triggered: already shown one-time guides
    public HashSet<int> Triggered { get; set; } = new();
    
    // Cooldown tracking: guide ID -> last shown time
    public Dictionary<int, DateTime> LastShown { get; set; } = new();
    
    // Dismissed: player manually closed (do not show again)
    public HashSet<int> Dismissed { get; set; } = new();
}
```

### 触发检查逻辑

```csharp
public bool CanTrigger(Player player, int guideId, GuideConfig config)
{
    var state = player.GuideState;
    
    // Check if already triggered (one-time)
    if (config.OneTime && state.Triggered.Contains(guideId))
        return false;
    
    // Check if dismissed
    if (state.Dismissed.Contains(guideId))
        return false;
    
    // Check cooldown
    if (config.Cooldown > 0 && state.LastShown.TryGetValue(guideId, out var last))
    {
        if (DateTime.Now - last < TimeSpan.FromSeconds(config.Cooldown))
            return false;
    }
    
    // Check condition
    return config.Condition(player);
}
```

## 客户端需求 | Client

### 显示组件

| GuideType | UI组件 | 行为 |
|-----------|--------|------|
| SystemPrompt | 模态对话框 | 阻塞输入，需要操作 |
| ContextualHint | 浮动文本 | 5秒后自动消失，非阻塞 |
| Waypoint | 地图标记 + 箭头 | 到达或关闭后消失 |
| Achievement | 弹出横幅 | 3秒后自动消失 |
| SoftPromotion | 底部面板 | 可关闭，有操作按钮 |
| Tutorial | 步骤覆盖层 | 高亮UI元素，等待操作 |

### 待确认问题

1. **模态 vs 非阻塞**：客户端能否支持阻塞（SystemPrompt）和非阻塞（ContextualHint）两种显示？

2. **路径点渲染**：路径点如何显示？屏幕边缘箭头？地图标记？两者都有？

3. **教程覆盖层**：分步教程时，客户端能否高亮特定UI元素？

4. **本地化**：标题/内容是发送本地化文本，还是语言键由客户端查找？

5. **队列处理**：如果多个引导同时触发，是排队还是按优先级？
