# 技能系统 | Skill System


---

## 系统架构

### 存储层次

```text
Life(生物) → Part(身体部位) → Skill(技能) → Movement(动作)
```

### 核心机制

* **部位绑定**：技能存储在身体部位，不同部位学习不同技能类型
* **自动分配**：新增身体部位时自动获得基础攻击技能  
* **条件驱动**：技能使用需满足等级、前置、身体等条件
* **技能聚合**：角色属性和战斗能力基于所有技能统计

## 条件系统

### 支持的条件类型

| 条件前缀 | 格式示例 | 描述 | 检查对象 |
|----------|----------|------|---------|
| 身体 | `身体·手` | 身体部位要求 | 角色是否拥有该部位 |
| 技能 | `技能·基础格斗·10` | 前置技能及等级要求 | 角色技能等级 |
| 手持 | `手持:Weapon·Sword` | 手持物品标签要求 | 手持物品的标签 |
| 手持 | `手持:体积×3` | 手持物品属性要求 | 手持物品的属性值 |

**手持条件支持两种检查模式**：
- **标签检查**：`手持:标签名`，检查手持物品是否包含指定标签
- **属性检查**：`手持:属性名×值`，检查手持物品的属性是否满足条件
  - 支持属性：`体积`、`重量`、`价值`
  - 支持运算符：`×`（≥）、`<`、`>`、`==`、`<=`、`>=`

### 条件检查逻辑

**所有条件必须同时满足**：
```
if (身体条件 AND 技能条件 AND 手持条件) {
    允许使用
} else {
    禁止使用
}
```

**示例**：
```
条件字符串："技能·基础剑术·5,手持:Weapon·Sword"
检查：
1. 角色是否学会"基础剑术"且等级≥5？
2. 角色手持物品是否包含"Weapon·Sword"标签？
```

```
条件字符串："手持:体积×3"
检查：
1. 角色手持物品的体积是否≥3？
```

## 自动分配机制

### 分配规则

**触发时机**：角色获得新身体部位时

**分配技能**：基础格斗（默认基础技能）

**防重复检查**：
1. 检查角色自身是否已拥有该技能
2. 检查角色所有部位是否已拥有该技能
3. 只有完全不存在时才分配

**初始水平**：
- 经验：0
- 等级：1

---

## 技能整合

### 整合规则

**角色可用技能** = **自身技能** + **所有身体部位技能**

**重复处理**：
- 相同技能取等级最高者
- 不累加，只保留最强版本

**示例**：
```
角色自身：基础格斗 Lv5
左手部位：基础格斗 Lv3
右手部位：基础格斗 Lv7
结果：角色使用基础格斗 Lv7
```

## 战斗接口

### 技能调用

**存储位置**：技能存储在身体部位

**条件检查目标**：所属角色（不是部位本身）

**手持物品匹配**：
- 支持手持物品标签检查（如`手持:Weapon·Sword`）
- 支持手持物品属性检查（如`手持:体积×3`）
- 手持条件自动排除无手部位的角色（如无需额外添加`身体·手`条件）

**接口方法**：
```csharp
Life.GetAllSkills()  // 获取角色所有可用技能
Skill.CanUse(Life)   // 检查技能是否可用
Skill.Use(Life)      // 使用技能
```

**特殊效果说明**：

**Cleave（横扫群攻）**：
- 单次攻击命中多个目标（最多3个）
- 对场景内所有敌对角色造成伤害
- 刀法技能（横斩、旋风斩）使用此效果

**MultiHit（多段连击）**：
- 单次攻击连续打击多次
- 支持配置目标模式、段数规则、伤害修正
- 完整格式：`MultiHit:[段数]:[目标模式]:[伤害修正类型]:[修正值]`
- 简化格式：可省略后续参数，使用默认值

**MultiHit 配置参数**：

1. **段数规则**：
   - `N` - 固定段数，例如 `3`
   - `Min~Max` - 随机段数，例如 `3~6`
   - `Level/10` - 等级公式，段数=技能等级/10（向下取整）

2. **目标模式**（可选，默认SingleTarget）：
   - `SingleTarget` - 所有段数攻击同一目标
   - `RandomTarget` - 每段随机选择目标

3. **伤害修正类型**（可选，默认None）：
   - `None` - 无修正，每段100%伤害
   - `Decay` - 递减修正，首次即衰减，公式 `伤害 × 系数^(击数+1)`
   - `Ratio` - 固定比例修正
   - `Total` - 总伤控制修正

4. **修正值**（可选，根据修正类型）：
   - Decay: 衰减系数（如0.85表示第1击85%，第2击72.25%）
   - Ratio: 伤害比例（如0.6表示每段60%伤害）
   - Total: 总伤倍数（如1.5表示总伤害为普攻的150%，平均分配到各段）

**MultiHit 配置示例**：
- `MultiHit:2` → 固定2段，同一目标，无修正
- `MultiHit:2:SingleTarget:Ratio:0.6` → 2段，同一目标，每段60%伤害
- `MultiHit:3~6:SingleTarget:Total:1.5` → 3~6段随机，同一目标，总伤150%
- `MultiHit:Level/10:SingleTarget:Decay:0.85` → 段数随等级变化，同一目标，85%衰减

**Cleave 配置格式**：
```
Cleave:[目标数]:[伤害修正类型]:[修正值]
```

**Cleave 配置示例**：
- `Cleave:2:Ratio:0.6` → 2个目标，每个60%伤害
- `Cleave:3~6:Total:1.5` → 3~6个目标，总伤150%
- `Cleave:Level/10:Ratio:0.4` → 目标数随等级变化，每个40%伤害

### 动作配置

**Movement** 是 **Skill** 的具体招式/动作：

**刀法示例**（群体攻击）：
```
Skill: 刀法
├── Movement 1: 横斩（1级解锁，Incise + Cleave:2:Ratio:0.6）
├── Movement 2: 旋风斩（30级解锁，Incise + Cleave:3~6:Total:1.5）
└── Movement 3: 雷霆斩（60级解锁，Incise + Cleave:Level/10:Ratio:0.4）
```

**剑术示例**（多段连击）：
```
Skill: 剑术
├── Movement 1: 二连斩（1级解锁，Incise + MultiHit:2:SingleTarget:Ratio:0.6）
├── Movement 2: 乱舞斩（30级解锁，Incise + MultiHit:3~6:SingleTarget:Total:1.5）
└── Movement 3: 一念斩（60级解锁，Incise + MultiHit:Level/10:SingleTarget:Decay:0.85）
```

## 属性计算

### 战力贡献

$$Power = 基础属性 + \sum(技能属性加成) + 装备属性加成$$

**技能属性加成**：
- 每个技能根据等级提供属性加成
- 高等级技能提供更多加成
- 技能越多，战力越高

### 专精计算

$$专精值 = \sum(指定类型技能等级)$$

$$最小值 = 1$$

**示例**：
```
剑术专精 = 基础剑术Lv5 + 高级剑术Lv10 + 神剑术Lv3 = 18
```

## 招式解锁机制

### 解锁规则

**技能等级解锁**：技能中的招式根据技能等级逐步解锁

**配置格式**：
```
movements: "招式A:等级,招式B:等级,招式C:等级"
```

**示例**：
```
剑术: "二连斩:1,乱舞斩:30,一念斩:60"
刀法: "横斩:1,旋风斩:30,雷霆斩:60"
```

**规则说明**：
- 等级1解锁第一个招式
- 达到指定等级后自动解锁对应招式
- 未解锁的招式不会出现在战斗技能选择池中
- 可用招式随技能等级提升而增加

## 技能学习

### 学习方式

根据新世界观，技能学习方式调整为：

**1. 任务奖励**
- 完成城市任务获得技能书
- 完成遗迹探险获得技能传承

**2. NPC教学**
- 城市导师传授基础技能
- 特殊NPC传授高级技能

**3. 自我领悟**
- 战斗中实战经验积累
- 达到条件自动领悟新技能

**4. 遗迹传承**
- 探索遗迹发现古代技能
- 解读铭文学习失传技能

### 技能分类（新世界观）

**基础技能**：
- 基础格斗（所有种族）
- 基础狩猎（所有种族）
- 基础生存（所有种族）

**种族专属**：
- **Lemurian**：灵性技能、魔法技能
- **Atlantean**：力量技能、战斗技能
- **半兽人**：种族天赋技能（如Naga的毒素、Horus的飞行）

**武器技能**：
- 剑术、刀术、棍术、拳术等（通用）

**特殊技能**：
- 从遗迹中学习的远古技能
- 从旧神封印中领悟的禁忌技能

## 数据结构

### Skill存储

```csharp
Skill {
    Config: Config.Skill           // 技能配置
    Exp: int                       // 经验值
    Level: int                     // 等级
    NextExp: int                   // 升级所需经验
    Movements: Movement[]          // 技能包含的动作
}
```

### Movement存储

```csharp
Movement {
    Config: Config.Movement        // 动作配置
    Damages: HashSet<Damage>       // 伤害类型集合
    Effects: HashSet<Effect>       // 效果类型集合
}
```

## 与其他系统的关系

### 种族系统

- 不同种族身体部位不同
- 不同部位可学习的技能不同
- 种族天赋影响技能效果

### 战斗系统

- 战斗时选择可用技能
- 技能条件检查
- 技能效果应用

### 装备系统

- 武器类型影响技能可用性
- 装备属性影响技能效果

### 生产系统

- 生产技能（工程）通过生产活动获得经验提升等级
