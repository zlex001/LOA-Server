# 登录系统（Authentication）

负责玩家身份验证、账号管理与登录流程，包括传统登录、游客快速开始和设备绑定机制。

## 通用模块（Agent）

**职责边界**：
- 监听网络事件（`Client.Event.Login`、`Client.Event.QuickStart`、`Client.Event.InitializeRandom`）
- 路由到具体的业务逻辑模块（`Login.cs`、`Register.cs`、`Device.cs`）
- 管理HTTP路由（`/api/authentication/device`、`/api/authentication/ips` 等）

**主要方法**：
- `OnLogin()` - 传统登录请求处理器
- `OnQuickStart()` - 游客快速开始请求处理器
- `OnInitializeRandom()` - 角色随机属性生成处理器

---

## 传统登录（Login）

**流程**（`Login.Do`）：
1. 版本校验 - 检查客户端版本是否匹配
2. 账号查询 - 检查账号是否存在
3. 区服验证 - 检查玩家是否属于当前区服
4. 密码验证 - 校验密码正确性
5. 在线检查 - 若已在线则踢出旧连接
6. 创建角色 - 在玩家保存的地图位置创建角色实体
7. 完成登录 - 调用 `CompleteLogin()` 发送游戏数据

**首次登录**：
- 账号不存在 → 自动调用 `Register.Start()` 启动注册流程
- 客户端收到 `Initialize` 协议，显示角色创建界面
- 玩家输入角色名 → 发送 `InitializeConfirm` → `Register.CompleteRegistration()`

---

## 游客快速开始（QuickStart）

**设计目标**：降低新玩家进入门槛，跳过登录注册界面，直接进入游戏体验。

**触发方式**（`Login.QuickStart`）：
- 客户端发送 `Net.Protocol.QuickStartRequest` 协议
- 包含字段：`device`（设备ID）、`version`、`platform`、`language`

**流程**：
1. **版本校验** - 检查客户端版本
2. **设备绑定检查** - 通过 `Device.GetBoundAccountId(device)` 查询设备是否已绑定账号
3. **自动登录** - 若设备已绑定，直接调用 `Login.Do()` 自动登录已有账号
4. **创建游客账号** - 若设备未绑定，创建新账号：
   - 账号ID：`Traveler{timestamp}{4位随机数}` 例如 `Traveler173890123456789`
   - 角色名：`Traveler{4位随机数}` 例如 `Traveler1234`
   - 密码：空字符串（可后续设置）
5. **随机生成属性** - 调用 `Register.GenerateRandomCharacter()` 生成年龄、性别、类别、属性点、出生地图
6. **完成注册** - 直接调用 `CompleteRegistration` 逻辑，跳过角色创建UI
7. **设备绑定** - 调用 `Device.Bind(device, accountId)` 绑定设备与账号
8. **完成登录** - 调用 `CompleteLogin()` 发送游戏数据

**设备绑定机制**：
- 同一设备再次 `QuickStart` → 自动登录已有账号（不创建新账号）
- 换设备 → 需要账号密码登录，或重新 `QuickStart` 创建新账号
- 游客转正功能：暂未实现，未来可支持绑定邮箱/手机、设置密码

**账号命名规则**：
- 使用 `Traveler`（旅者）符合游戏世界观
- 时间戳确保账号ID唯一性
- 4位随机数确保角色名可读性

## 注册流程（Register）

**两阶段注册**：
1. **账号创建** - `Register.Start()` 创建 `Data.Database.Player`，发送 `Initialize` 协议
2. **角色确认** - 玩家输入角色名 → `Register.OnConfirm()` → `CompleteRegistration()`

**随机属性生成**（`GenerateRandomCharacter`）：
- 年龄：初始玩家年龄 × 游戏时间倍率
- 性别：随机（Female / Male）
- 类别：随机（Atlantean / Lemurian）
- 属性点：5个属性（HP/MP/LP/Strength/Agility），总和100，随机分配（1-25）
- 出生地图：从 `Data.SpawnPoint.GetRandomInitialMap()` 随机选择

**数据同步**（`SyncPlayerDataToDatabase`）：
- 将运行时玩家数据（位置、部位HP、资源）同步到数据库
- 调用时机：注册完成时、登出时

---

## 登录完成（CompleteLogin）

**协议发送顺序**（关键）：
1. **设备绑定** - `Device.Bind(device, accountId)`
2. **恢复宠物** - `RestoreCompanions(player)`
3. **教程初始化** - `Tutorial.Instance.Start(player)` - **发送 UILock 协议**
4. **准备数据** - 可行走区域、资源、场景、角色列表、UI本地化
5. **发送 Home 协议** - 包含游戏主界面所需的所有数据
6. **发送 WorldMap 协议** - 世界地图数据
7. **发送 ScreenAdaptation 协议** - 屏幕自适应参数

**为什么 Tutorial.Start() 必须在 Home 之前**：
- Tutorial.Start() 会调用 `Display.Agent.InitializeUILock()` 或 `UnlockAllHomePanels()`
- UILock 协议告诉客户端哪些面板可见
- 若 Home 协议先到达，客户端不知道UI锁定状态，会显示所有面板

## 设备管理（Device）

**设备注册**（`ProcessRegister`）：
- HTTP路由：`/api/authentication/device`
- 客户端发送设备ID和平台信息
- 服务器创建或更新 `Data.Database.Device` 记录
- 返回服务器列表供玩家选择

**设备绑定**（`Bind`）：
- 登录成功后，将设备ID与玩家账号ID关联
- 存储在 `Data.Database.Device.player` 字段
- 用于 `QuickStart` 时自动登录

**绑定查询**（`GetBoundAccountId`）：
- 输入：设备ID
- 输出：绑定的玩家账号ID（若无绑定则返回 null）

## 网络协议

### QuickStartRequest（客户端→服务器）
```json
{
  "device": "string",      // 设备唯一标识
  "version": "string",     // 客户端版本（如 "1.2.0"）
  "platform": "string",    // 平台（如 "Windows", "iOS", "Android"）
  "language": "string"     // 语言（如 "ChineseSimplified"）
}
```

### LoginResponse（服务器→客户端）
```json
{
  "code": 0,               // 0=Success, 1=PasswordError, 2=AppVersionUnfit, 3=UnsafeAccount
  "message": "string"      // 错误信息（成功时为空）
}
```

---

## 错误处理

| 错误码 | 含义 | 原因 |
|--------|------|------|
| `Success` | 登录成功 | - |
| `PasswordError` | 密码错误 | 密码不匹配或区服不匹配 |
| `AppVersionUnfit` | 版本不匹配 | 客户端版本过旧或过新 |
| `UnsafeAccount` | 账号不安全 | 账号ID或密码包含非法字符 |

## 数据库字段

### Data.Database.Player
- `Id` - 账号ID（唯一标识）
- `text["Pw"]` - 密码（QuickStart账号为空字符串）
- `text["Name"]` - 角色名
- `text["Gender"]` - 性别（Female / Male）
- `text["Category"]` - 类别（Atlantean / Lemurian）
- `text["ServerId"]` - 区服ID（用于区服隔离）
- `record["Age"]` - 年龄（游戏内天数）
- `grade[Attribute]` - 属性点分配（HP/MP/LP/Strength/Agility）
- `pos[]` - 出生地图坐标
- `time["Register"]` - 注册时间
- `time["SignOut"]` - 上次登出时间

### Data.Database.Device
- `Id` - 设备唯一标识
- `Platform` - 平台枚举（iOS / Android / Windows 等）
- `PreferredLanguage` - 首选语言
- `player` - 绑定的玩家账号ID

---

## 设计原则

1. **安全性**：
   - 账号ID和密码校验（6-20字符，安全字符集）
   - 区服隔离（玩家只能登录所属区服）
   - 在线状态检查（防止多端登录）

2. **用户体验**：
   - QuickStart 跳过繁琐的注册流程
   - 设备绑定实现"一键登录"
   - 版本校验避免不兼容客户端连接

3. **可扩展性**：
   - 游客账号与正式账号数据结构一致
   - 预留"游客转正"功能扩展点（绑定邮箱/手机）
   - 支持跨设备登录（通过账号密码）
