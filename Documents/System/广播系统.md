# 广播系统

广播系统负责向玩家发送游戏消息，通过不同频道控制传播范围，支持多语言和动态文本替换，确保每个玩家看到个性化的消息内容。

系统采用统一的 API 设计，所有频道都支持从简单文本到复杂彩色文本的完整表现能力。

消息使用占位符模板，如"{sub} 拾取了 {item} × {count}"，系统会自动替换为实际内容，最终显示为"张三 拾取了 铁剑 × 1"。

**发送者** 即触发广播消息的角色，通常是执行操作的玩家或 NPC。

**接收范围** 由选择的频道决定，从个人专属到全服广播覆盖不同的玩家群体。

**动态替换** 通过{玩家名}、{物品名}、{数量}等占位符让一个文本模板适用于多种情况。

**多语言支持** 根据玩家的语言设置自动选择对应的文本版本。

**事件驱动** 当游戏中发生拾取、装备、战斗等事件时自动触发相应的广播。

**文本片段** 即组成消息的各个部分，可以是多语言文本 ID、彩色文本、原始字符串等不同类型内容的组合。

**统一接口** 所有频道都使用相同的参数结构，简化开发使用并提供一致的功能能力。
- **触发角色** 执行操作的角色，决定消息的发送位置和范围基准
- **消息片段** 构成完整消息的各个部分，可包含多语言文本ID、彩色文本、原始字符串
- **占位符数据** 用于替换消息模板中占位符的实际内容，如玩家名、物品名、数量等

## 道具数量显示

广播消息中的道具显示遵循智能数量规则，避免重复显示数量信息，确保文本简洁清晰。

**主体道具** 即消息中操作的主要目标道具，通常使用item作为占位符名称。

**上下文道具** 即消息中提及的容器、接收者等其他道具，使用bag、obj、equipment等占位符名称。

**数量占位符** 即文本模板中的{count}占位符，用于显示操作涉及的数量。

系统会根据消息中是否存在数量占位符，自动决定道具是否显示数量后缀：

**有数量占位符时**，主体道具不显示数量后缀，避免重复：
- 文本模板："{sub}将{count}个{item}递予了{obj}。"
- 实际显示："张三将3个【包子】递予了李四。"

**无数量占位符时**，主体道具显示自身持有数量：
- 文本模板："{sub}将{item}放进了{bag}。"
- 实际显示："店主将【游戏币×90】放进了【行囊】。"

**上下文道具**始终不显示数量后缀，保持简洁：
- 文本模板："{sub}将{count}个{item}放进了{bag}。"
- 实际显示："张三将3个【包子】放进了【行囊】。"（行囊不显示×1）

这个规则确保了无论道具在哪个位置出现，显示的数量信息都是最合理和直观的。

## 系统 | System

系统频道用于发送个人专属消息，只有指定的玩家能看到。这类消息通常是关于玩家自身状态变化的重要提醒。

典型应用场景包括等级提升通知、获得奖励提示、技能学习成功、余额变更通知等。这些都是只与特定玩家相关，不需要其他人知道的信息。

消息发送时需要指定具体的目标玩家，系统会确保消息只出现在该玩家的界面上。

## 本地 | Local

本地频道用于当前地图内的广播，该地图内的所有玩家都能看到相关消息。这是最常用的频道类型，适合大部分玩家行为的广播。

主要用于玩家的日常操作广播，比如拾取物品、装备更换、物品交易、跟随状态变更等。这些行为虽然是个人操作，但其他玩家看到这些信息有助于了解周围的活动情况。

消息会发送给触发事件的地图内所有在线玩家，让大家都能及时了解当前区域发生的事情。

## 战斗 | Battle

战斗频道专门处理战斗相关的消息，包括伤害数值、技能释放、状态变化等战斗信息。

这类消息对战斗的参与者非常重要，能帮助玩家了解战斗进程和伤害情况。同时观战的玩家也能通过这些信息了解战斗的激烈程度。

战斗频道特别适合使用复杂的文本组合功能：
- **技能名称**：根据技能类型显示不同颜色
- **伤害数值**：暴击伤害用特殊颜色突出显示  
- **状态提示**：不同的战斗状态用对应颜色区分

系统会向战斗发生地图内的所有玩家发送这些消息，确保相关人员都能看到战斗信息。

## 全服 | All

全服频道是覆盖范围最广的广播方式，所有在线玩家都会收到消息。由于影响范围很大，这个频道主要用于重要公告和特殊事件。

典型使用场景包括服务器维护通知、重大活动开启、特殊奖励发放等需要所有玩家都知道的重要信息。

使用这个频道需要谨慎考虑，避免过于频繁的全服广播影响玩家体验。

## 传闻 | Rumor

传闻频道用于播报游戏世界中的传闻和特殊事件，营造丰富的游戏氛围。

这类消息通常具有一定的随机性和趣味性，比如某个知名 NPC 的动向、特殊事件的发生等。这些信息不一定与玩家直接相关，但能增加游戏世界的生动感。

消息会发送给所有在线玩家，但频率相对较低，避免信息过载。

## 自动化 | Automation

自动化频道主要服务于游戏的脚本和自动化功能，发送与自动化操作相关的提示信息。

当玩家使用自动战斗、自动采集等功能时，系统会通过这个频道发送相关的状态提示和结果通知。

消息通常只发送给使用自动化功能的玩家本人，确保提示信息的针对性。

## 私人 | Private

私人频道在当前系统中保留但暂未实际使用，为将来可能的私聊或密信功能预留接口。
