# 颜色设计规范

## 设计原则

1. **语义优先** - 调用方使用颜色类别（枚举），而非具体颜色值
2. **集中管理** - 所有颜色定义在 `Utils.Text.Colors` 枚举和 `color` 字典中
3. **禁止硬编码** - 业务代码中禁止出现 `#RRGGBB` 或 `<color=...>` 硬编码
4. **一致性** - 相同语义的内容必须使用相同颜色类别

---

## 调用方式

```csharp
// 正确：使用颜色类别
Utils.Text.Color(Utils.Text.Colors.Success, "操作成功")

// 错误：硬编码颜色
$"<color=#00FF00>操作成功</color>"
```

## 辅助函数

| 函数 | 用途 | 返回值 |
|------|------|--------|
| `Color(Colors key, string text)` | 为文本添加颜色标签 | 带颜色标签的字符串 |
| `LevelColor(int stage, string text)` | 根据等级返回品质颜色 | 带颜色标签的字符串 |
| `GetDamageColor(double ratio)` | 根据伤害比例返回颜色 | Colors枚举值 |
| `GetDangerRatioColor(int used, int max)` | 根据使用率返回危险度颜色 | 十六进制字符串 |

## 待补充类别

以下语义目前缺少专用颜色，暂时复用现有类别：

| 语义 | 场景 | 当前方案 | 建议 |
|------|------|----------|------|
| 奖励/祝福 | 升级、领取经验、获得奖励 | 复用 `Success` | 可新增 `Reward` |
| 成就 | 成就解锁、里程碑 | 复用 `Quality6` | 可新增 `Achievement` |
| 系统公告 | 重要全服公告 | 复用 `ChannelSystem` | 可新增 `Announce` |

---

## 重要规则

1. **禁止硬编码** - 业务代码中不得出现具体颜色值
2. **语义匹配** - 选择颜色时优先考虑语义而非视觉偏好
3. **新增审批** - 新增颜色类别需评估必要性，避免无序扩张
4. **文档同步** - 新增或修改颜色后，必须同步更新本文档
