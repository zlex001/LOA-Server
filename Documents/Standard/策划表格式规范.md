# 策划表格式规范

## 分隔符

### 逗号`,`：并列列表

用于分隔同类型的多个项目：

```text
剑,盾,头盔
基本剑法,基本内功,基本轻功
房间A,房间B,房间C
```

### 分号`;`：多项内容

用于分隔包含键值对的多个项目：

```text
技能:偷窃;物品:潜影手套×1;经验:3000
料理点×1;木箱×1;木桌×1;店主:(100~100)×1
银子:1000;药草:5;铁剑:1
```

## 运算符

### 冒号`:`：键值对

用于表示键和值的对应关系：

**简单键值对：**

```text
银子:1000
体质:10
Level:5
```

**多参数配置（连续冒号连接）：**

```text
MultiHit:3:SingleTarget:Decay:0.8
```

说明：
- 第一个冒号前是配置名（如 `MultiHit`）
- 后续冒号按顺序连接各参数
- 参数可以是数值、枚举值、范围表达式
- 参数可以省略，使用默认值

**示例：**

```text
MultiHit:2                                    # 2段连击，其他参数默认
MultiHit:(3~6):RandomTarget                   # 3~6段随机，随机目标
MultiHit:3:SingleTarget:Decay:0.8             # 3段，同一目标，递减修正0.8
MultiHit:2:SingleTarget:Ratio:0.4             # 2段，同一目标，每段40%伤害
```

### 箭头`=>`：映射关系

用于表示左侧到右侧的映射或转换关系：

```text
地图=>角色配置
埃利都-牢房=>灰泥之狐·萨尔:80×1
```

### 括号`()`：范围框定与优先级控制

括号在配置中有两种用途：

**用途1：框定范围表达式**

范围表达式必须用括号框起来：

```text
(5~10)           # 数值5到10
(1~100)          # 数值1到100
等级:(10~20)     # 等级10到20
店主:(100~100)   # 固定值100（最小=最大）
```

**用途2：控制逻辑运算优先级**

改变逻辑运算的优先级和明确作用域：

```text
(A | B) & C                      # 先计算或，再计算且
!(A & B)                         # 非运算作用于整个表达式
(身体:手 | 身体:爪子) & Level>=5   # 先判断身体条件，再判断等级
```

### 波浪号`~`：范围表达式

用于表示最小值到最大值的范围：

```text
(5~10)           # 数值5到10
(1~100)          # 数值1到100
```

**说明：**
- 必须用括号 `()` 框起来
- 适用于所有类型的范围：数值、等级、属性等
- 当最小值=最大值时，表示固定值
- 符合数学区间表示习惯

### 乘号`×`：数量

表示数量，语义由所在表决定：

```text
村长×3        # 3个村长
木箱×5        # 5个木箱
羊:(5~9)×1    # 1只5-9级的羊
```

**在不同表中的含义：**
- **地图表（Map）**：每个地图实例的数量（微观）
  - `羊×3` = 每个Forest地图都有3只羊
- **场景表（Scene）**：整个场景的总数量（宏观）
  - `白鹿×10` = 埃纳达平原整个场景共10只白鹿

### 短横线`-`：层级关系

用于表示父子层级关系：

```text
埃纳达平原-岸边              # 场景-地图
埃利都-农夫的家              # 场景-地图
TheftQuest-粮仓陷阱          # 任务线-任务节点
```

### 星号`*`：权重

用于表示概率权重：

```text
房间A*0.6,房间B*0.4          # 60%概率A，40%概率B
草地*3,树林*1                # 75%草地，25%树林
```

### 百分号`%`：百分比

用于表示概率百分比：

```text
100%×1            # 100%概率掉落1个
银两:50%×10~20    # 50%概率掉落10-20个银两
丹药:25%×1~3      # 25%概率掉落1-3个丹药
```

**Drop标签应用示例：**

```text
Drop:铁矿石:30%×1~3                          # 30%概率掉落1-3个铁矿石
Drop:银矿石:40%×1~2                          # 40%概率掉落1-2个银矿石
Drop:金矿石:50%×2~5                          # 50%概率掉落2-5个金矿石
Drop:铁矿石:30%×1~3;银矿石:10%×1~2           # 多个掉落物配置（分号分隔）
```

**说明：**
- 每次攻击Item时触发掉落判定
- 概率独立计算，多个掉落物可同时掉落
- 数量范围采用随机算法，包含最小值和最大值

**语法结构：**
- `%` 表示概率
- `×` 连接概率和数量
- `~` 表示数量范围
- `50%×10~20` 表示：50%的概率掉落10-20个
- 注意：数量范围不使用括号，括号仅用于等级范围如`:(5~10)`

**地图/场景characters字段应用示例：**

characters字段用于配置地图或场景中生成的角色（Life）和物品（Item）。

**Item（物品）格式：**

```text
木箱                    # 1个，100%概率（所有参数默认值）
木箱×3                  # 3个，100%概率
铁矿石:50%×2            # 2个，50%概率
铁矿石:50%×2~5          # 2-5个（随机数量），50%概率
```

**Life（生物）格式：**

```text
农夫:(100)              # 100级（固定等级），1只，100%概率
农夫:(100~120)          # 100-120级（随机等级），1只，100%概率
农夫:(100~120)×3        # 100-120级，3只，100%概率
农夫:(100~120)×2~5      # 100-120级，2-5只（随机数量），100%概率
农夫:50%×3:(100~120)    # 100-120级，3只，50%概率（完整格式）
```

**说明：**
- Item无需等级，Life必须指定等级范围`(等级)`或`(最小~最大)`
- 默认值：概率=100%，数量=1
- 完整格式：`名称:概率%×数量:(等级范围)`
- Life的完整格式使用两个冒号连接参数，符合位置参数规范
- 括号仅用于等级范围，数量范围不使用括号
- 多个角色用逗号`,`分隔：`木箱×3,铁矿石:50%×2,农夫:(100~120)×1`

### 斜杠`/`：动态计算表达式

用于动态计算公式中的除法运算：

```text
Level/10                    # 技能等级除以10
Level/10+2                  # 技能等级除以10再加2
```

**特性：**
- 这是**算术运算符**，用于表达式计算，区别于键值对、分隔等其他运算符
- 主要用于等级动态计算
- 支持 `Level` 关键字表示技能等级
- 计算结果向下取整
- 常用于段数、目标数量等随等级变化的配置

**应用示例：**
```text
MultiHit:Level/10:SingleTarget:Decay:0.85     # 段数=技能等级/10
Cleave:Level/10:Ratio:0.4                     # 目标数=技能等级/10
```

### 方括号`【】`：范围标记

用于标记作用域或范围值：

```text
【0】     # scope为0
【2】     # scope为2
【3】     # scope为3
```

### 方括号`[...]`：嵌套配置

用于包围嵌套的子配置内容：

```text
酒架×1[尚清村的酒:100%×1]                      # 酒架容器内100%掉落1个酒
宝箱×1[银两:50%×(10~20),丹药:25%×(1~3)]       # 宝箱容器内的多个掉落配置
```

**嵌套配置内的分隔符规则：**
- 内部的多个同类项使用**逗号**`,`分隔（嵌套层级内简化）
- 示例：`[物品A:概率×数量,物品B:概率×数量,物品C:概率×数量]`
- 说明：嵌套配置`[...]`是特殊场景，内部用逗号以避免与外层分号混淆

## 逻辑符

### `&`：且关系

表示必须同时满足多个条件：

```text
身体:手&武学:基本内功×10      # 需要手部 且 基本内功>=10级
Level:10&Event:主线完成       # 等级>=10 且 完成主线事件
```

### `|`：或关系

表示满足任一条件即可：

```text
身体:手|身体:爪子              # 手部 或 爪子
武器:剑|武器:刀                # 剑 或 刀
```

### `!`：非关系

表示条件取反（没有/不满足）：

```text
!(Skill:偷窃)                  # 没有偷窃技能
!(Event:主线完成)              # 主线未完成
!(Item:钥匙)                   # 没有钥匙
```

### 运算符优先级

逻辑运算符的优先级从高到低：

1. `!`（非）- 最高优先级
2. `&`（且）- 中优先级
3. `|`（或）- 最低优先级

**说明：**

```text
A | B & C           # 等价于 A | (B & C)，先计算且再计算或
!A & B              # 等价于 (!A) & B，先计算非再计算且
!A | B & C          # 等价于 (!A) | (B & C)
```

**建议：**
- 当混用多个运算符时，建议使用括号明确优先级，提高可读性
- 避免编写过于复杂的嵌套表达式
- 括号的用途详见"运算符"章节

**组合使用：**

```text
!(Skill:偷窃) & Level>=5      # 没有偷窃技能 且 等级>=5
!(Event:任务A) | Event:任务B  # 任务A未完成 或 任务B已完成
!(Level>=10)                  # 等级小于10
```

**嵌套使用：**

```text
!(身体:手 | 身体:爪子)         # 既没有手 也没有爪子
!(Skill:偷窃 & Level>=5)      # 不满足"有偷窃技能且等级>=5"
```

## 运算符优先级总览

在复杂表达式中，运算符按以下优先级顺序解析（从高到低）：

### 1. 括号 `()`

- 最高优先级
- 用于框定范围表达式和改变运算优先级
- 示例：`(5~10)`、`(A | B) & C`

### 2. 非逻辑 `!`

- 逻辑运算最高优先级
- 作用于紧邻的表达式
- 示例：`!A & B` 等价于 `(!A) & B`

### 3. 除法 `/`

- 算术运算符
- 用于动态计算公式
- 示例：`Level/10+2`

### 4. 数量运算符 `×`

- `×` 表示数量
- 地图表：每个地图实例的数量（`村长×3`）
- 场景表：整个场景的总数量（`白鹿×10`）

### 5. 概率与权重 `%` 和 `*`

- `%` 表示概率百分比
- `*` 表示权重
- 示例：`50%×(10~20)`、`草地*0.5`

### 6. 且逻辑 `&`

- 逻辑运算中等优先级
- 表示必须同时满足
- 示例：`A & B | C` 等价于 `(A & B) | C`

### 7. 或逻辑 `|`

- 逻辑运算最低优先级
- 表示满足任一条件
- 示例：`A | B & C` 等价于 `A | (B & C)`

### 8. 键值对 `:`

- 连接键和值
- 示例：`银子:1000`、`MultiHit:3:SingleTarget`

### 9. 映射关系 `=>`

- 表示左侧到右侧的映射
- 示例：`地图=>角色配置`

### 10. 分隔符 `,` 和 `;`

- 最低优先级
- `,` 分隔并列列表
- `;` 分隔多项内容
- 示例：`剑,盾,头盔`、`技能:偷窃;经验:3000`

**注意事项：**

- 当混用多个运算符时，建议使用括号明确优先级，提高可读性
- 避免编写过于复杂的嵌套表达式
- 嵌套配置 `[...]` 内部的运算符优先级规则保持一致

## 符号使用注意

### 乘号`×`与短横线`-`的区别

- `×`（乘号）：数量运算符
- `-`（短横线）：层级关系运算符

**正确示例：**
```text
埃纳达平原-岸边:(10~20)       # 场景-地图，等级范围10~20
白鹿×10                       # 场景表：整个场景10只白鹿
村长×3                        # 地图表：每个地图3个村长
```

**错误示例：**
```text
埃纳达平原×岸边               # ❌ 会被误解为数量
村长-2                        # ❌ 会被误解为层级关系
```

### 短横线`-`与波浪号`~`的区别

- `-`（短横线）：层级关系运算符
- `~`（波浪号）：范围表达式符号

**正确示例：**
```text
埃利都-路:(5~10)              # 场景-地图，等级5~10
```

**错误示例：**
```text
埃利都-路:5-10                # ❌ 混淆了层级符和范围符
```

### 全角方括号`【】`与半角方括号`[...]`的区别

- `【】`（全角方括号）：范围标记，用于标记scope等作用域值
- `[...]`（半角方括号）：嵌套配置，用于包围容器内的子配置

**正确示例：**
```text
【0】                                        # 全角：标记scope为0
酒架×1[尚清村的酒:100%×1]                   # 半角：容器嵌套配置
```

**错误示例：**
```text
[0]                                          # ❌ 应使用全角【0】
酒架×1【尚清村的酒:100%×1】                 # ❌ 应使用半角[...]
```

**说明：**
- 全角`【】`键盘输入：中文输入法下按`[`或`]`
- 半角`[...]`键盘输入：英文输入法下按`[`或`]`
- 两种括号功能完全不同，不可混用

## 组合使用示例

### 奖励配置

```text
技能:偷窃;物品:潜影手套×1;经验:3000;银子:(500~1000)
```

### 地图配置

```text
料理点×1;木箱×2;木桌×1;店主:(100~100)×1;顾客:(50~80)×3
```

### 场景配置

```text
白鹿:(10~20)×10;铁矿×3;珍稀草药×5
兔:(2~5)×2;鹿:(7~12)×2;清水×1
```

**说明：**
- 可以指定等级范围：`生物名:(最小等级~最大等级)×数量`
- 也可以省略等级（物品无等级概念）：`物品名×数量`
- 使用分号`;`分隔多个配置项
- 场景表的数量表示**整个场景的总数量**

### 条件表达式

```text
Level:10&Event:主线洪水之后
(身体:手|身体:爪子)&武学:基本内功×5
```

### 权重列表

```text
草地*0.5,树林*0.3,岸边*0.2
```
